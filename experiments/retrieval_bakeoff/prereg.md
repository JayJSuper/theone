# PREREG — 记忆检索核 bake-off（余弦 vs 频谱 vs HRR）

**状态**: 预注册（开跑前冻结）| **执行**: CC | **日期**: 2026-06-13
**纪律**: 采纳门槛=在冻结基准上赢过余弦基线；输了不进架构。诚实预测在先。

## 0. 假说（来自七路径审计）
频谱幅匹配（路径①）、HRR 全息（路径④）在某些检索 regime 上优于余弦基线。给三核一场公平对照。

## 1. 三个检索 regime（冻结）
- **R1 语义**：目标 = 查询 + 小噪声（σ=0.3）。测平凡近邻检索。
- **R2 移位**：目标 = 查询的随机循环移位；干扰项 = 随机向量。测移位不变性。
- **R3 绑定**：查询 = bind(role, filler)；候选 = 各 (role_j, filler_j) 绑定对；目标 = 共享 filler 者。测结构化检索（HRR 用 unbind 清理记忆，余弦/频谱用原始向量）。

## 2. 设置（冻结）
- 维度 d=64；每 regime：N=200 候选键，M=50 查询，20 seeds。
- 指标：top-1 命中率（目标是否被排第一）。
- 三核：cosine（基线）、spectral、hrr（R3 用 unbind 协议，R1/R2 退化为原始向量相似）。

## 3. 判据（冻结）
- 某核在某 regime **胜**：top-1 命中率显著高于余弦（20 seeds 配对，≥15 更优）。
- 报告 3×3 命中率矩阵 + 每 regime 胜者。
- **预期结论**：无单核通吃——余弦赢 R1，频谱赢 R2，HRR 赢 R3 → 支持"多范式器官"论（不同核是不同器官，按 regime 路由）。若余弦通吃，则七路径的检索声称在本测试下不成立，记录。

## 4. 诚实预测（开跑前写死）
余弦赢 R1（语义近邻它最优）；频谱赢 R2（已在单测验证移位不变）；HRR 赢 R3（绑定结构是它的主场）。**若任一预测落空，如实报告，不救。** 关键诚实点：即使三核各赢一个 regime，也只证明"互补"，**不证明**它们在真实记忆任务上比"余弦+足够数据"更划算——那需要端到端任务，不在本轮。
