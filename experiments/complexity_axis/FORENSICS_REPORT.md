# 失败模式逆向分析 — gpt-5.1 在组合爆炸下到底怎么崩的

**日期**: 2026-06-13 | **依据**: Q-C21 攻击② + 守门员补充提醒一 | **方法**: $0 重算（SCM 由 (k,i) 确定性生成）
**真实度**: 【已验证】，含一次 CC 自我证伪（见 §3）。

## 1. 问题
gpt-5.1 在 k=5 时 45/50 错答、k=6 时 42/50 错答。守门员问：是系统性近似（可校准/可提示修复），还是随机崩溃（需更大预算）？并建议查它是否用了某种"捷径"（只调 k-1 个混杂 / 退化成不调整 / 常数猜测）。

## 2. 结果（去混淆后）
| | k=5 | k=6 |
|---|---|---|
| 偏差 mean(pred−truth) | +0.0247（轻微高估） | +0.0042 |
| 与真值相关 corr | +0.58 | +0.71 |
| 匹配"不调整(观测)"捷径 | 5/45 (11%) | 0/42 (0%) |
| 匹配"只调 k-1 混杂"捷径 | 5/45 (11%) | 3/42 (7%) |
| 错答 std | 0.137 | 0.030 |
| 真值 std | 0.052 | 0.041 |

**裁定：SCATTERED degradation（散射式退化）——没有干净的捷径可归因。** 两个候选捷径（不调整 / 丢一个混杂）的命中率都≈随机水平（7-11%）。gpt-5.1 不是采用了某个可命名的近似策略，而是"算到预算耗尽，然后给出散布的估计"。

## 3. ⚠️ CC 自我证伪（这条最重要）
**我的第一版分析过度声明了"k=6 崩成常数 0.5"。** 自查发现：**高 k 时真值本身就挤在 0.5 附近**（k=6 真值 std 仅 0.041、范围 [0.408,0.613]）——因为平衡混杂平均化趋向 0.5。所以"错答聚集在 0.5"大部分是真值分布的假象，不是 gpt-5.1 的"最大熵默认"策略。去混淆判据（错答 std 须显著小于真值 std）后，k=6 不构成"常数猜测"signature。**收回过度声明。**

## 4. 诚实结论
1. **没有可校准的系统性捷径**：错答不匹配"不调整"也不匹配"丢一个混杂"（均≈随机）。提示工程/校准无简单修复对象——因为没有系统性偏差可纠。
2. **k=5 的错误幅度大于真值信号**：错答 std 0.137 ≫ 真值 std 0.052——在一个真值本就集中的区域里，gpt-5.1 反而给出更散的猜测。
3. **退化是"渐进散射"而非"断崖归零"**：保留正相关（0.58/0.71），方向大致对、精度全失。这与"推理链在预算内逐步失真"一致。
4. **对凭证哲学的支撑**：正因为失败是散射而非系统性，**外部无法靠校准识别这些错答**——这恰是"每个答案需带可验证凭证"的论据：你不能靠观察输出分布发现它错了。

## 5. 与 v0+规模+k 轴叙事的衔接
三命门证明"组合爆炸下 LLM 崩"。本分析补上"**怎么崩**"：不是优雅地退回简单启发式，而是散射式失真且无警告——这比"45/50 错"信息量大得多，且强化了凭证的必要性。**经 CC 自我证伪去除了一个假 signature。**
