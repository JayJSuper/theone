# 附录 — Claude 跨基座补测（悬崖 k4/k5）

**日期**: 2026-06-13 | **依据**: Q-C21 攻击④守门员建议 | **模型**: claude-opus-4-8（fable-5 access-gated）
**协议**: 与 k 轴正式跑相同的 k=4/k=5 各 50 题（种子复现）| AM-007 评分 | **真实度**: 【已验证】

## 1. 结果：四基座在悬崖处的表现
| 基座 | k=4 (2⁴=16) | k=5 (2⁵=32) |
|---|---|---|
| 引擎 C | **1.00** | **1.00** |
| **claude-opus-4-8** | **1.00** | **0.10**（45 协议失败） |
| gpt-5.1 | 0.66 | 0.08 |
| gemini-3-flash（12节点参照） | — | — |
| deepseek-v4-flash | 0.06 | 0.00 |

## 2. 两个发现
1. **"可终止性是架构限制"声称增强**：第四个基座（claude，又一个不同模型家族）**同样在 k=5 崩溃**（0.10）。四个基座（deepseek/gemini/gpt-5.1/claude）跨三个模型家族，**全部在 2⁵=32 配置处崩**。这不是某家的偶然——是 LLM 类方法面对组合爆炸的**共同架构边界**。
2. **claude 的崩溃签名 = deepseek 型（协议失败）而非 gpt-5.1 型（自信错答）**：claude k=5 是 45/50 **协议失败**（推理链耗尽交白卷），与 deepseek 同类；gpt-5.1 是自信错答。**两种崩溃签名跨基座都存在**——有的模型"算不完就认怂"，有的"算错还自信"。**后者（自信错答）正是凭证哲学最需要防的**，而 claude/deepseek 的"认怂"反而更安全（至少它没给你错数字）。

## 3. claude 的 k=4 满分意味着什么
claude 和引擎在 k=4 都 1.00，而 gpt-5.1 仅 0.66、deepseek 0.06——说明 claude-opus-4-8 的有效推理预算在四个 LLM 里最高，**把悬崖推到了 k=4 之后**。但 k=5 它也崩——**更强的推理只是推后悬崖一档，不能消除它**（守门员提醒3 在第四个基座上再次兑现）。

## 4. 对声称范围的最终裁定
原声称"LLM 类方法在组合爆炸下崩溃"——**经四基座跨三家族验证，可以发表，不需再收窄**。悬崖位置随基座推理预算在 k=4~k=5 间浮动，但 2⁵=32 配置处**无一幸免**。引擎全程 1.000。
