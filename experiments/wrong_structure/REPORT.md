# REPORT — 赌注①生态效度·第三步：错结构鲁棒性与「凭证认证什么」的诚实边界

**日期**: 2026-06-14 | **无 API，纯计算** | **复现**: `python experiments/wrong_structure/run.py`
**真实度**: 【已验证】，含一条**必须随结论同行的关键限制**。

## 0. 一句话
引擎 + 凭证保证「**给定结构下**计算精确」。本实验测当**调整集错了**（漏掉混杂）会怎样：在弱多混杂族里引擎**优雅退化**且高 k 仍胜 LLM；但漏掉一个**强**混杂会产生 **0.30 的静默偏差**——**凭证认证的是计算、不是结构**。这是赌注①最诚实的边界，也是为什么「因果发现」是下一前沿。

## 1. 系统扫描（复用悬崖 SCM，弱多混杂族，k=2..6，每格 80 实例）
后门调整在混杂子集 S 上：`do_S(X=1)=Σ_s Π_{i∈S}P(U_i=s_i)·P(Y=1|X=1,S=s)`。S=全部→精确 do（已 sanity 校验 <1e-9）。

| k | 不调整偏差(全混杂) | 漏1个·均值 | 漏1个·最坏 |
|---|---|---|---|
| 2 | 0.0361 | 0.0300 | 0.0425 |
| 4 | 0.0253 | 0.0170 | 0.0293 |
| 6 | 0.0128 | 0.0091 | 0.0173 |

- **优雅退化**：漏 1 个混杂的偏差 < 不调整（漏全部）；都随 k 收缩（混杂越多每个越不关键 + 高 k 真值集中 0.5）。
- **高 k 仍胜 LLM**：k=5,6 时漏一个混杂的引擎偏差 ~0.009–0.022，**仍小于** LLM 高 k 推理崩溃 ~0.05–0.10。即**即便结构不完美，引擎在高 k 仍优于 LLM 端到端**。
- **重要 caveat**：这里偏差小，是因为本族**每个混杂都弱**（随机 CPT + 多混杂稀释 + 近 0.5 集中）。**不是普适的"鲁棒"声称。**

## 2. 决定性反例：漏掉一个**强**混杂 → 0.30 静默偏差
构造经典强混杂（U=1 时 X 极可能、Y 极高；U=0 反之）：
- 真 do(X=1) = **0.5750**
- 漏掉该（唯一）混杂的估计 = **0.8750**（观测 P(Y|X=1)）
- **静默偏差 = 0.3000** —— 远大于 LLM 高 k 误差。

偏差大小**只由混杂强度决定**，无上界。引擎照样给出一个带「计算精确」凭证的**自信而错误**的答案——和高 k 的 LLM 自信错答**同构**，只是错因不同（LLM=算不动；引擎=结构喂错）。

## 3. 诚实结论（这是凭证哲学的真实边界）
> **认知凭证认证的是「在给定因果结构上，计算是精确、可第三方复算的」——它不认证「这个结构本身是对的」。** 结构正确是**前提**，不是凭证的保证项。

- 漏弱混杂 → 优雅退化（偏差 ∝ 该混杂强度，本族小）。
- 漏强混杂 → 灾难性**静默**偏差（0.30+），凭证不报警。
- 与 NOTE-002（前门近违反静默偏差）同源：**结构/假设层的错误，是引擎+凭证体系的共同盲区**。凭证 limits 字段必须写明「computation-exact, structure-assumed」。

## 4. 对路线的含义
三步生态效度梯度已成：
1. 精确 CPT + 已知 DAG → 引擎碾压（三命门）。
2. **估计** CPT + 已知 DAG → 优势存活 k≥4（AM-018）。
3. 已知 CPT + **错结构** → 优雅退化（弱）/ 静默灾难（强）；**凭证不保结构**（本报告）。

**剩下的唯一前沿**：DAG 未知，需从数据学结构（`docs/NEXT_FRONTIER_causal_discovery.md`）。本报告正是它的最强动机——**因为结构错误是静默且无上界的，所以"如何可靠地得到结构"是赌注①能否落地真实世界的决定性问题。**
