# PREREG — 洛书路由假说基准赛

**状态**: 预注册（开跑前冻结）| **执行**: CC | **日期**: 2026-06-13
**纪律**: 对假说敬畏，用基准说话。判据冻结不挪；赢了进架构，输了进《美丽失败录》。诚实预测在先（防事后诸葛）。

## 0. 假说（来自 Jay + 七路径文档）
洛书 3×3 幻方（任意行/列/对角之和恒为 15）是一种天然平衡结构，**可能**作为负载均衡/路由结构优于平凡基线。本实验给它一场公平对照赛。

## 1. 任务（冻结）
将 N 个键路由到 9 个桶（worker），度量负载均衡与抗热点能力，对照四种路由器。

## 2. 路由器（冻结）
- **luoshu**: 桶 = MAGIC_FLAT[hash(key) % 9]，MAGIC_FLAT=[4,9,2,3,5,7,8,1,6]（幻方按行展开）
- **modulo / round-robin**: 桶 = key_index % 9（均匀流的最优基线）
- **random**: 桶 = 均匀随机（种子固定）
- **consistent_hash**: 标准一致性哈希（160 虚拟节点/桶，工业标准）

## 3. 访问模式（冻结，三种）
- **uniform**: N 个独立均匀键
- **skewed (Zipf, s=1.2)**: 少数热键（测抗热点）
- **sequential_burst**: 键成段连续到达（测相邻键是否被打散，避免相邻 worker 同时过载）

## 4. 指标（冻结）
- **负载不均衡** = max_bucket_load / mean_bucket_load（越接近 1 越好）；同时报方差。
- **2D 平衡**（洛书独有的可能优势）：若键带 (行,列) 二维标签，行方向与列方向负载的最大不均衡——测幻方是否同时平衡两维。
- N ∈ {90, 900, 9000}；每条件 20 seeds（随机/键生成）。

## 5. 判据（冻结）
- luoshu **胜**：在某访问模式上不均衡显著低于**全部**基线（配对，20 seeds 中 ≥15 更优）。
- luoshu **持平**：与 round-robin 在均匀流上统计无差（预期结果）。
- luoshu **败**：劣于任一基线。
- 三种情况全部如实报告。

## 6. 诚实预测（开跑前写死）
**预测 luoshu 在一维负载均衡上≈round-robin（无优势）**——round-robin 对均匀流已是最优，幻方的"行列和=15"性质对一维路由无直接帮助。**唯一可能的真优势在 2D 平衡**（行列同时均衡），这是幻方性质的直接推论；若此处也无优势，则诚实判定"洛书路由假说在本测试下未显优势"，入《美丽失败录》，不强行救。
