# REPORT — 洛书路由假说：未显优势，入《美丽失败录》

**日期**: 2026-06-13 | **预注册**: 48c54b4（诚实预测在先，判据未改）
**复现**: `python experiments/luoshu_routing/run.py` | results_sha=44a2ceb781f17283 | CPU $0
**真实度**: 【已验证·零结果】。预注册预测兑现：洛书在本测试下无路由优势。

## 1. 结果（N=9000，中位不均衡，1.0=完美均衡）
| 访问模式 | luoshu | round-robin | random | consistent-hash |
|---|---|---|---|---|
| uniform | 1.053 | **1.000** | 1.048 | 1.080 |
| skewed (Zipf) | 2.148 | **1.000** | 1.048 | 2.808 |
| sequential_burst | 1.053 | **1.000** | 1.048 | 1.086 |

2D 平衡（uniform）：luoshu=1.019 vs 最优其它=1.019（持平）。

> **⚠️ 订正（2026-06-13，何老师反馈触发，自捕第5次）**：此 2D 指标**无效**——它累加的 `(hash(key)%3, hash(key)//3%3)` 标签来自 key 哈希、**不看路由器选的桶**，四家必然并列，等于没测路由决策。故下方"2D 无优势 / 唯一优势处也没出现"作废。改用**全枚举**直接测幻方真性质（对角约束）见 `diagonal_value.py`：对角线作检错冗余确有增益（拒绝 89% 骗过行列检查的排列），**但**对单/局部错误行列校验已足够、对角线零增益——优势仅限罕见威胁模型。详见 `docs/luoshu/RESPONSE_TO_HE_TEACHER.md`。

## 2. 判定（冻结判据）
- uniform / skewed / sequential：**luoshu_loses**（round-robin 全胜）——**有效，成立**（该指标用了路由决策）
- ~~2D 平衡：no_2d_advantage~~ —— **作废**（指标无效，见上方订正）

## 3. 诚实判读（含对假说的公平辩护）
1. **round-robin 全胜，但它是"到达序"路由**（按 idx%K），不按键——它**无法**在分布式场景按键复查桶位（需要状态）。所以对洛书（无状态键哈希）不完全公平。
2. **在公平对照组（无状态键路由器）内**：洛书在 skewed 上（2.15）确实**优于工业标准 consistent-hash（2.81）**——但这不能归因于幻方性质，**任何固定置换**大概率给出类似结果；且 random 仍更优。
3. **关键否证**：幻方独有的"行列对角和=15"性质，在 2D 平衡指标上**未带来任何可测优势**（与最优基线逐位持平）。这是假说的核心卖点，而它落空了。

## 4. 结论
**洛书作为路由结构，在本基准下未显示任何源于其幻方性质的优势。** 假说不成立（在路由这一应用上）。按章程入《美丽失败录》，不强行救。

## 5. 预设转向（章程"不否定"原则）
- 洛书可能在**别的**应用上有价值（如纠错码、采样设计、注意力稀疏模式），但那是不同假说、需各自的基准——不因本次失败而否定，也不因情怀而续测路由。
