# REPORT — 半结构化野生文本：一个被自捕的双 bug 伪迹，与赌注①边界的进一步收窄

**日期**: 2026-06-13 | **冻结**: REJ-002（撤回 AM-015） | **真实度**: 【已验证·含 CC 第4次自捕（commit 后撤回）】
**一句话**: 本实验最初给出"The One k=1 精确 vs gpt-5.1 漂移 MAE 0.065"的漂亮结果，已**提交并登记为 AM-015**——随后我自查发现它是**两个相消 bug 的伪迹**，全部撤回。修正后的真相对赌注①是收窄的：**低 k 时 gpt-5.1 自己就能正确做后门边缘化，引擎无优势。**

## 1. 原始（已撤回）结果
数字埋入 prose、同前端隔离设计，n=24：raw 严格命中 1/24、MAE 0.065、最大 0.211；The One 24/24、MAE 0。我据此登记 AM-015 称"赌注①核心主张的公平隔离证明"。**这是错的。**

## 2. ⚠️ 自捕：两个相消 bug（REJ-002）
- **Bug 1（叙事生成）**：`narrate()` 把两个**非对角** Y 单元格写反。item 0 引擎 truth=0.6536 用真实 P(Y\|X=1,U=0)=**0.43**；但叙事告诉 gpt-5.1"有干预的人里非混杂组占 **28%**"（那其实是 P(Y\|X=0,U=1)）。**叙事与 truth 不一致。**
- **Bug 2（引擎喂数）**：`the_one_estimate()` 的单元格映射有一处**对称反转**，恰好抵消 Bug 1。
- **后果**：gpt-5.1 读到被错标的叙事，算出 0.5816——**这对它被告知的题是完全正确的**，被我误判为"漂移"；The One 靠两 bug 相消凑出 truth。整个 k=1 对照**毫无意义**。

## 3. 根因诊断（如何抓到的）
做 k=2 扩展时出现矛盾信号：bug-free 的 k=2（叙事⟺truth 经离线验证一致）里 **raw gpt-5.1 反而 18/18 近精确（MAE 0.0001）**。"更难却更准"不合理 → 逐项手算 item 0 → 发现 gpt-5.1 的 0.5816 = 用**叙事百分数**做的正确后门 ≠ 引擎 truth 0.6536 → 锁定 narrate 写反单元格 → 再查出 the_one_estimate 的相消反转。
- 又做诊断 k=1b（仅把干扰项措辞换成无数字）：raw 与原 k=1 **逐项字节相同** → 证明 gpt-5.1 在这些题上确定性、且与干扰项措辞无关，排除"干扰项污染"假说，坐实是叙事错标。

## 4. 修复后的真实结果（bug-free）
两处 bug 修复 + 加入 `_selfcheck()`（叙事⟺truth⟺引擎三向一致性断言，启动即跑）后重跑：

| 指标 | RAW gpt-5.1（脑内边缘化） | The One（引擎边缘化） |
|---|---|---|
| k=1 严格命中 ±0.005 | **24/24 (1.000)** | 23/24 (1.000) |
| k=1 MAE | **0.0000** | 0.000000 |
| k=1 抽取有效率 | — | 95.8%（24 题干净跑） |
| k=2 严格命中 | **18/18 (1.000)** | 17/17 (1.000) |
| k=2 MAE | **0.0001** | 0.000000 |
| k=2 抽取有效率 | — | 70.8%（疑部分受 API 截断，待干净重跑） |

（混杂陷阱真实：mean_gap k1=0.094 / k2=0.062；raw 从不落在观测值上（trap=0）。注：k=1 抽取有效率在不并发烧 API 时回升到 95.8%——早期 62.5% 系限速截断的伪低；故"抽取是瓶颈"一说在 k=1 **不成立**，需对 k=2 干净重跑才能下结论。）

## 5. 修正结论（对赌注①收窄、诚实）
> **低 k（1–2）时，数字若正确陈述在自然语言里，gpt-5.1 自身就能正确做后门边缘化（k=1 24/24 精确、k=2 MAE 0.0001）。引擎在此 regime 无可测优势。**

- 引擎**已证**的优势，**仅存于高 k 组合爆炸悬崖（结构化输入）**（三命门、悬崖实验，依然成立、未受影响）。
- "引擎在 prose 上的低 k 边缘化优势"**不存在**。
- 高 k 在 prose 下能否复现引擎优势是**真正的未决问题**：抽取在 k=1 干净跑达 95.8%（不构成瓶颈），但 k=2 待干净重跑确认；且高 k prose 叙事本身的可陈述性/可抽取性是新的变量。这条线的结论尚未到手，**不得提前宣称**。

## 6. 流程教训（为什么没早抓到）
原 k=1 自测把 SCM 的 `v-dict` **直喂引擎**、绕过了叙事文本——它验证了"引擎吃对数=truth"（平凡为真），却**从未验证"叙事是否陈述了对的数"**。bug 正藏在"叙事⟺truth"这条没被测的缝里。**已固化补救**：`_selfcheck()` 现在解析叙事文本里的百分数、用正确后门复算、并要求 == 引擎 truth，启动即断言。
