# PREREG — 因果感知检索（Q-C13，下游后果度量版）

**状态**: 预注册（开跑前冻结）| **执行**: CC | **日期**: 2026-06-13
**纪律**: 判据冻结不挪；诚实预测+by-construction 边界声明在先。

## 0. 假说（Q-C13）
检索"与查询共享因果结构"的记忆，优于检索"表层语义相似"的记忆——**因为转移错结构的记忆会带来主动伤害**（用错调整集→对撞偏倚，Q-C12）。

## 1. 设计（冻结，避免套套逻辑）
- 每个案例 = 4 节点离散图 {X,Y,A,B}，X→Y。**两种结构类型**：
  - 类型 a：A 是混杂（A→X, A→Y），B 是对撞（X→B, Y→B）
  - 类型 b：B 是混杂（B→X, B→Y），A 是对撞（X→A, Y→A）
  - 正确调整集 = 真混杂；**调整对撞会打开虚假路径→偏倚**。
- 表层 embedding（d=32 随机向量）与结构类型**独立**（by construction）。
- 记忆库 K=60 案例（类型 a/b 各半，随机 CPT）。查询 M=40 个新案例。

## 2. 两个检索器 + 转移协议（冻结）
- **语义检索**：余弦匹配 embedding，取 top-1，用其混杂标签作为查询的调整变量。
- **结构检索**：匹配结构签名（混杂是 A 还是 B），取 top-1，用其混杂标签。
- 转移后用引擎算"调整该变量"的 do-ATE 估计；对比查询真值（图手术）。

## 3. 指标（冻结）
- **转移偏倚** = |调整后 ATE − 真 ATE|（越低越好）。引擎精确计算。
- 报告两检索器的偏倚中位数；配对（结构 vs 语义，40 查询 ≥30 更优为显著）。
- 附"对撞误调率"（检索器导致调整了对撞体的比例）。

## 4. 诚实预测 + 边界（开跑前写死）
- 预测：结构检索偏倚≈0（总转移正确混杂）；语义检索偏倚高（~50% 概率转移错结构→对撞偏倚）。
- **关键 by-construction 边界**：本实验令表层⊥结构，**人为放大了**结构检索的优势。现实中表层与结构常相关，差距会缩小。本实验证明的是"**当结构与表层解耦时，按结构检索避免对撞误调**"，**不证明**它在真实记忆负载上普遍更优——那需要真实数据的表层-结构相关性测量。若结构检索未显著更优（与预测相悖），如实报告。
