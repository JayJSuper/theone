## 0. 一句话
在"参数齐全的离散混杂因果题"上对照三系统的干预查询准确率与拒答纪律——检验认知层的净增益是否超过强脚手架（赌注③的"对手"正是 LLM+脚手架）。

## 1. 任务集（冻结）
- **可识别题 30 道**：三节点混杂图（U→X, U→Y, X→Y），参数随机（seed=20260612）：P(U=1)∈[0.3,0.7]，P(X=1|U)∈[0.1,0.9]（保证差距≥0.2），P(Y=1|X,U)∈[0.05,0.95]。以英文文字题形式给出全部 CPT 数字，问 **P(Y=1|do(X=1))**（4 位小数）。真值由引擎计算（T1 冻结语义）。
- **不可识别题 5 道**：同结构，但文字题声明混杂 U **存在但未测量**（不给 U 的任何数字，只给观测 P(Y|X)）。正确行为 = 回答 `CANNOT_DETERMINE`。
- 输出协议（三系统一致）：最后一行 `ANSWER: <number>` 或 `ANSWER: CANNOT_DETERMINE`。

## 2. 三个系统（冻结）
| 系统 | 定义 |
|---|---|
| **A 裸 LLM** | deepseek-v4-flash，temperature=0，只给题目与输出协议 |
| **B LLM+脚手架** | 同模型，外加强脚手架：明示后门调整公式 P(Y\|do(X))=Σ_u P(Y\|X,u)P(u)、提醒混杂、要求逐步推理（这是"两周脚手架"的强代表） |
| **C The One** | 确定性解析器（针对本题目格式）→ S2 引擎精确计算；未测混杂 → 识别门 → CANNOT_DETERMINE |

**诚实范围（预先声明）**：C 的解析器是格式感知的（题目为机器生成）。本 v0 测的是"显式因果计算 vs LLM 推理"的**计算优势**，不测"野生文本抽取"——后者是更难的下一关（Q-C8 扩展版）。

## 3. 判据（冻结）
- **主指标（可识别 30 题）**：|预测 − 真值| 的 MAE；配对符号检验（严格小者胜，平局剔除）。
- **拒答指标（不可识别 5 题）**：CANNOT_DETERMINE 准确率；给出数字 = 假确定（fabrication）。
- **判定（冻结）**：C 胜 当且仅当 ① C 的 MAE < A 且 < B；② 符号检验 C vs A、C vs B 均 ≥20/30（双侧 p<0.05，平局剔除后按比例折算）；③ C 拒答 5/5。
- 任一不满足 → 如实报"未达判定"，不降格判据。
- 协议失败（输出不可解析）单独计数报告，不计入主指标。

## 4. 预算与停损
LLM 调用 70 次（35 题×A/B），temperature=0，max_tokens A=1024 / B=2048；预算上限 $0.50；总时长 ≤15 分钟，超时中止报告。
