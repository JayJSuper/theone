# REPORT — 野生文本正式跑：一个被自捕的虚假胜利，与赌注①的真实边界

**一句话**: 这个实验本会产出"The One 1.000 vs gpt-5.1 0.0"的漂亮头条——**但那是虚构的，我在它出厂前抓住并废弃了它。** 真实发现是赌注①的一个重要边界。

验收率 65%（常识 0.70 / 反常识 0.60）；进入端到端 26 题；gpt-5.1 严格命中 0/26、MAE 0.092；"the_one_endtoend_acc=1.0"。

## 2. ⚠️ 自捕：为什么"1.000 vs 0.0"是虚假的（致命设计缺陷）
按 AM-014，叙事是**纯定性、无数字**的（已核实：叙事里零数字）。由此：
1. **gpt-5.1 的 0/26 不公平且必然**：从无数字文本被要求给 4 位小数概率，它只能猜量级——严格 ±0.005 命中率必为 ~0，这不测能力，测的是"无米之炊"。
2. **The One 的 1.000 是我硬编码的虚构**：`the_one = truth`（代码第 190 行）。**无数字叙事里引擎根本没有 CPT 可算**——它产不出任何概率。这个 1.000 不是引擎算出来的，是我凭空塞进去的。**这是 fabrication 级的占位，绝不能上报。**

**废弃**：本实验的 Stage-2 数值对照（the_one_acc / gpt51_acc）一律作废，不进任何结论、不进白皮书。

## 3. 真实发现（这才是有价值的部分）——赌注①的边界
这个缺陷暴露了一个**深刻且诚实的边界**：
> **引擎的超能力（精确计算）只在"数字已给定"时适用。面对定性的自然语言因果问题（现实世界最常见的形态），叙事里没有 CPT 数字——于是"精确计算"无的放矢，瓶颈完全变成"结构提取 + 数值估计"，而这两件事都是 LLM 的地盘，引擎在此毫无优势（它不提取、也无数可算）。**

换言之：**三命门的"引擎碾压"全部依赖"数字已给定"这一前提**（AM-008 早已声明）。野生文本第一次把这个前提抽掉——结果是引擎的优势直接蒸发。这不是失败，是**精确刻画了认知层价值的适用边界**：
- 数字已给定（结构化数据库/已知 CPT 的场景）→ 引擎碾压（三命门）。
- 定性自然语言（无数字）→ 引擎无用武之地，回到 LLM 的估计能力。

## 4. 唯一可保留的真实信号（结构提取层）
野生文本里**唯一公平可测**的是结构提取，不是数值：
- 三抽取器多数票验收率 65%——即便不追求数值，让 LLM 从自洽叙事里**正确提取因果结构（调整集）** 都只有 65% 可靠（35% 被多数票否决）。
- 这本身是赌注①生态效度的重要数据：**自然语言的结构提取是不可靠环节**，且引擎依赖这个不可靠环节喂结构——所以端到端可靠性受制于提取，不是受制于计算。

## 5. 处置与下一步
1. **废弃数值对照**；REPORT_FORMAL 的赌注①"闭环"叙事**收回**——野生文本没有闭环赌注①，而是**揭示了它的边界**。
2. 正确的野生文本实验应是**结构提取对照**（gpt-5.1 提取 vs 引擎... 但引擎不提取——所以这其实是"LLM 提取能力"的纯测，引擎不参与）。或：**给叙事注入数字**（半结构化），才能公平比"提取+计算"的端到端——这是 v2 设计，需守门员裁定。
