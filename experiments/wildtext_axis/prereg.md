# PREREG — 野生文本轴 PILOT：叙事生成管线可行性

**状态**: 预注册（开跑前冻结）| **执行**: CC | **日期**: 2026-06-13
**依据**: Q-C8（AM-006 五旋钮）、Q-C17①（AM-009 对撞惩罚 1/3 已冻结）、Q-C17②（协议含 CC 结构级回译修正案，**待守门员确认**——本 PILOT 同时作为该修正案的可行性物证）
**范围**: 本 PILOT 只测**生成管线的因果保真度**（叙事能否无损携带因果结构），**不测**被试系统的表现——那是正式跑。

## 1. 生成管线（冻结）
SCM 实例（变量取自冻结词库，三元组+干扰变量）→
- **L1 叙事**：S1（deepseek-v4-flash）改写为自然段落，**显式**陈述混杂的双向影响
- **L2 叙事**：混杂被提及（与处理、结果的关联以观察性语言出现），**因果角色不标注**
→ **双 LLM 独立抽取**（deepseek-v4-flash + gpt-5.1，跨基座强化独立性）：从叙事抽取 {treatment, outcome, confounders, adjustment_set}（JSON）→
- **交叉校验**：两抽取器核心字段一致？
- **结构级回译**（CC 修正案）：抽取结果与原始 SCM 的真实角色比对——treatment/outcome 匹配 + 调整集 = 真混杂集。L1/L2 为定性叙事，**不做 1e-6 ATE 回译**（无数值参数，Q-C17② 缺口）。

## 2. 词库（冻结，12 组三元组+干扰）
跨领域（健康/教育/农业/交通/商业/环境），见 pilot.py `BANK`。**已知局限（如实声明）**：本 PILOT 词库为真实世界合理结构，抽取器可能借助先验知识而非阅读文本——正式跑需加**反平衡设计**（随机化角色指派），列为 v1 设计点，不在本 PILOT。

## 3. 指标与判据（冻结）
- **验收率** = 双 LLM 一致 ∧ 结构回译匹配 的叙事比例；按 L1/L2 分层报告。
- 参照 Q-C17 阈值：≥85% → 管线可自动化；<85% → 暂停自动化，抽样分析失败原因（如实报告，不调阈值）。
- 样本：L1×12 + L2×12 = 24 叙事（每组三元组各一），seed=20260616。
- 预算：~72 次 LLM 调用，≤$0.5，≤30 分钟。

## 4. 诚实预测（写死）
L1（显式）验收率高（>85%）；L2（隐式）**可能低于 85%**——隐式混杂正是设计来为难抽取的。若 L2<85%，这不是管线失败，而是**L2 难度的早期标定数据**（被试系统将面对同样的难度）。两种结果都有用。
