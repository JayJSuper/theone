为什么"显式因果引擎 + LLM"优于"纯 LLM"——一张图说清

2026 年的顶级推理模型懂教科书因果，不怕无关信息——但当一道因果题需要边缘化的混杂配置数 2ᵏ 超过它的推理预算时，连旗舰也会崩。而显式因果引擎在 2ᵏ 上是平的：永远 1.000、毫秒级、每个答案带可验证凭证。

读图：横轴是"一道因果题需要在脑中边缘化的混杂配置数 2ᵏ"（对数刻度）。四个 LLM 跨三个模型家族，全部在 2⁵=32 配置处崩溃（旗舰把悬崖推后一档，但不能消除）。引擎绿线全程贴在 1.000。这就是"把组合爆炸的因果计算从推理链移出、交给确定性引擎"的价值。

一句话叙事

因果推理的正确性不是知识问题（LLM 懂因果），是计算可终止性问题——当边缘化组合数超过推理预算，再强的 LLM 也崩，而精确因果引擎不受组合爆炸影响。认知层的价值 = 把因果计算从推理链中移出，交给确定性引擎，并为每个答案附上可验证的凭证。

诚实边界（随结论同行）：三命门均在"因果图已知 + 参数齐全 + 结构化输入"条件下获得——证明的是"已知结构、高组合复杂度下精确计算优于有限预算推理"，不等价于"因果推理已解决"。两条野生文本实验进一步把适用域钉死：① 定性无数字叙事——引擎无 CPT 可算，优势不适用（WEAK-03）；② 即便把数字正确写进自然语言、在低组合复杂度(k=1–2)下，旗舰 LLM 自己就能把后门边缘化算到精确（k=1 24/24、k=2 MAE 1e-4），引擎在此 regime 无可测优势（SAT-02；过程中一个双 bug 伪迹已自捕撤回 REJ-002）。故引擎优势严格限于"高 k 组合爆炸 × 结构化输入"——即本图的悬崖区。所有数字可复现，失败/边界/撤回公开记录于《美丽失败录》。
