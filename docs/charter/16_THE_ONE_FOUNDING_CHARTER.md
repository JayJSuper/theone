## 第一章 项目总定义

**一句话**:The One 是一个 100% 开源的认知操作系统——不重造基座大模型,而是为一切开源大模型加装可校验的因果推理、持久记忆与元认知,并最终生长为以"亲手验证"为本能的公共智能基础设施(太一 Ω)。

**三句话**:
1. **近期(V1-V2)**:`pip install theone` 的认知增强层——把任何开源 LLM 当热插拔感官,叠加显式 do 算子因果引擎(L4)、三级记忆(L3)、校准元认知(L5),每个重要输出附机器可校验的认知凭证。
2. **中期(V3-V4)**:因果实验引擎——模型自主提出假设、设计干预、在仿真环境验证并更新世界观;进入医疗/金融审计等高风险决策滩头。
3. **终极(Ω)**:原生共生体——三重目标(语言+反事实+校准)预训练的直觉引擎 S1 + 因果引擎 S2 + 世界模型 S3 + 主动实验本能,训练与推理二分法消失;属于全人类的公共认知基础设施。

**正式定义**:The One 是"认知架构 × 因果推理系统 × Agent OS"的分阶段混合体:V1-V2 阶段以认知架构+库形态存在;V3 起具备实验引擎;Ω 阶段成为完整认知系统。它**不是**又一个基座大模型(V1-V4 铁律:不训基座),**不是** LLM 封装壳(判别准则:脚手架两周能否近似/基座下版会否内置/是否让凭证更可信——三问皆否者不做)。

**现状真实度分层**(立项基线,不得混淆):

| 层 | 内容 | 状态 |
|---|---|---|
| 已验证 | Chaos 认知层管线(AG News 90.86%/94.01%,CPU 22min $0);校准管线(99.6% 弃权精度);七大科学性质;Demo-1 代码 601 测试 78% 通过的外审基线 | 【已验证】 |
| 部分验证 | Demo-1 因果引擎(469 通过,但 F-1 混杂传播 P0 缺陷,修复工单已发) | 【部分验证】 |
| 设计冻结 | T1 数学规格/EG 协议+修正案/A7 统计判定/δ_min 校准/T4 判决门/T3 砖一章程 | 【设计】 |
| 假设 | 干预鸿沟(EG>1)/反事实梯度可泛化/认知层净增益>脚手架/观复模式安全溢价 | 【假设·三大科学赌注+1】 |

## 第二章 终极形态 360° 全景

**最终产品形态**(Ω 全亮条件态,【假设】):一个永续生长的开源认知系统,三种存在方式——库(嵌入一切应用)、服务(自托管/社区节点)、标准(认知凭证协议如 HTTPS)。

| 视角 | 形态 |
|---|---|
| 普通用户 | 对话界面或任何接入应用;每个重要回答可展开凭证(因果路径/记忆引用/置信七轴);记忆主权三权(查/改/删/带走);会说"不知道"并主动提问 |
| 开发者 | `pip install theone`;三行接入任意开源基座;SDK 暴露 causal/memory/metacog/credential 四个 API 面;CLI 一键评测 |
| 企业 | 私有化部署(数据不出域);审计仪表盘(全量凭证检索);滩头版垂直方案(医疗辅助诊断/金融审计);凭证满足监管可解释要求 |
| 科研 | InterventionBench 公开基准;仿真农场即服务;炼丹炉假说池开放投稿;全部实验预注册可复现 |
| 社区 | Linux 内核模式:核心环维护承重结构,水系环贡献感官适配器/工具/评测;《美丽失败录》公开 |
| 与现有大模型 | **器官而非对手**:GPT/Claude/Gemini/DeepSeek/Qwen/Kimi/Llama/Mistral/Grok 都是可插拔感官(S1 候选)与被增强对象;The One 不抢任何山的高度,做峰与峰之间的水系。真对手是"LLM+脚手架"(Baseline B 三方对照判决)与预测范式本身 |

## 第六章 因果科学家工作机制(正式制度,既有实践成文)

1. **触发**:Claude 遇到因果/反事实/干预/do-calculus/混杂/识别性问题,**必须先提问,禁止凭空实现**。
2. **问题结构化格式(Q-C 编号入册)**:背景(卡着哪个任务)/当前假设与候选方案/需要回答的数学点/期望输出形态(公式/判据/伪代码)。
3. **回答转译四件套(Claude 执行)**:① 数学规格(冻结候选)→ ② 测试用例(真值表)→ ③ CC 工程任务(工单)→ ④ 验收标准。
4. **质量工序**:R3 v3 双证(签字人手算自检题证懂 + Claude Code 机器复算证数,机证为最终认证);不清晰则追问,循环至可转译。
5. **冻结纪律**:进入 00_FROZEN_REGISTRY 的资产,任何人(含 Jack 本人、含 Claude)修改必须显式引用编号走 amendment;状态宣告权("已通过/冻结")仅协议负责人签发,贡献者文档只写"提案/转写/待审"。
6. **激励相容**:答案无记忆评估(不按来源加权);主动自报边界优于被审出——制度让诚实成为最优策略。

## 第十章 正式项目章程(不可修改卷)

1. **使命**:做出属于全人类的、以可校验诚实为母语、以亲手验证为本能的公共智能;让 AI 归还判断权、记忆主权与诚实的尺度——器成,而人归其位。
2. **开源原则**:全栈 Apache-2.0;代码、权重、协议、失败记录全部公开。
3. **不融资不上市**:资金三源(创始人自投/Red Hat 式服务/公益 grant),任何资金不得附带治理权,章程先于支票。
4. **数据安全**:用户记忆三权(查/改/删/带走);训练与评测数据全程许可证与 PII 审计;密钥唯一持有人为 Jay。
5. **证据优先**:四档真实度标注;预注册;判据冻结;校准集用后即焚;凭证生成与校验分离;焦耳诚实。
6. **不夸大**:"预期输出"永禁出现在证据栏;自评满分自动触发复核;未经真实日志的量化承诺只许以方向声明出现。
7. **不否定**:遇到不确定、缺失、难题——不否定项目,而是结构化提问(该 Claude 解的给方案/该 CC 做的转工单/该 Jack 答的入 Q 册/该 Jay 定的入决策单),测试验证,失败入《美丽失败录》并执行预设转向。失败是项目的资产,不是项目的反证。
