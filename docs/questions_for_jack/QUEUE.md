## Q-C10 ⭐ 高优先 — 协议失败是否计为"错"（正式跑命门公平性）
- **背景**: Baseline B v1 图规模轴正式跑中。PILOT 发现裸 LLM 与脚手架 LLM 在 12 节点图上的失败几乎全是**推理链耗尽 token 预算（4096）交不出答案**，而非算错。我们当前把"无答案"计为错。
- **数学点**: 这是否构成对 LLM 的不公平判定？(a) 应否给"预算加倍重试"作为敏感性分析？(b) 还是说"在固定预算下交不出可验证答案"本身就是一个合法且重要的能力差异（成本/可终止性维度）？(c) 若重试，重试预算的上限如何预注册才不算 p-hacking？
- **期望输出**: 判定规则裁示 + 是否要求"预算加倍敏感性跑" + 重试协议的防滥用约束。

## Q-C11 — 图规模轴的混杂结构控制
- **背景**: v1 规模轴用随机 DAG（强制 X→Y + 一个观测混杂 U* + 随机填充父集）。节点增多时，后门调整集大小、是否出现对撞/中介都在随机变化。
- **数学点**: 我们想隔离的是"图规模"，但随机结构混入了"识别难度"这个隐变量。是否应固定调整集基数（如恒为单后门），只让无关节点数增长？还是分层报告（按真实调整集大小分桶）？哪种更干净地分离"规模"与"识别复杂度"？
- **期望输出**: 结构控制方案（固定 vs 分层），及"规模"的操作定义建议。

## Q-C12 — 野生文本轴的标注与评分客观性
- **背景**: Q-C8 已定义野生文本 L0-L3 难度旋钮。但"L2 隐式混杂"的因果结构需要人/机判定，存在主观性。
- **数学点**: 如何保证"标准答案"客观？(a) 由我们从已知 SCM **正向生成**叙事（结构已知，但叙事自然度受限）？还是 (b) 取真实文本**反向标注**（自然但标注有争议）？(c) 评分时，模型答案与标准调整集的"部分正确"如何计分（全有全无 vs 集合 F1）？
- **期望输出**: 生成 vs 标注的取舍建议 + 部分正确的评分规则。

## Q-C13 — 记忆检索核三方对照的因果相关性

- **期望输出**: 是否值得设一个"因果感知检索"基线；若有，度量的数学草案。

## Q-C14 — 前门准则与工具变量的识别门扩展
